The Command List

This is your complete reference guide for using the RCSO Bot. We focus on making the system super sleek and easy. Below you'll find detailed explanations for every available command, organized by feature category.

Comprehensive Feature Overview

The RCSO Bot is designed to be the administrative backbone of your department. It seamlessly manages time, finances, and personnel records, ensuring accuracy and freeing up command staff to focus on leadership and training. Our core features include:

Automated Shift Logging
Modal Reporting (Tickets/Arrests)
FTO & Training Management

1. Shift Management & Activity

These commands are your daily tools for managing duty status. Using them ensures your time is accurately tracked for pay and promotion eligibility.

/shift_panel

Purpose: The central and easy way to manage your duty status. This command brings up an interactive button panel where you click "START SHIFT (10-8)" or "END SHIFT (10-7)". Starting a shift automatically applies the ON_DUTY role and opens a private log thread for your reference.

Message Content: A button panel with a current time stamp and status confirmation.

/status_update [new_status]

Purpose: Quickly log changes in your activity status during your shift (e.g., stopping a pursuit, going 10-6, vehicle maintenance). This command sends a message to your shift thread, keeping your log detailed without manual typing.

Usage Example: /status_update 10-6 break for 15 mins

Message Content: A simple confirmation message that your status has been logged in the shift thread.

2. Reporting & Records

These commands are critical for logging formal police actions. They all use simple pop-up forms (modals) to make data entry fast and structured, ensuring every event is tracked.

/report_panel

Purpose: Opens an interactive panel that lets you choose which type of report you want to file (Arrest, Citation, Incident). Each selection launches the correct modal form.

Message Content: A button panel listing all report types.

/arrest_log (via panel)

Purpose: Generates a full arrest report (Booking). The modal requires fields like Suspect Name, Charges, and a detailed Narrative. The submitted report is logged to the RECORDS channel and requires supervisor approval via buttons.

Message Content: A clean embed summary of the arrest details, tagged with a unique case number.

/citation_log (via panel)

Purpose: Logs traffic citations, warnings, or vehicle equipment orders. The form ensures consistency in reporting vehicle/driver information and violation codes. This process eliminates paper forms entirely.

Message Content: A formatted embed showing the ticket number and violation summary.

/incident_report (via panel)

Purpose: For logging non-arrest, non-citation events, such as high-speed pursuits, property damage, or officer-involved incidents that need documentation for internal review.

Message Content: An embed confirming the report submission and listing the assigned report number.

3. FTO & Training

These tools are essential for the Field Training Officer (FTO) program, simplifying session logging and tracking cadet progress.

/training_panel

Purpose: Opens an instructor panel with buttons to start a new training session or access controls like sending results or canceling the session.

Message Content: A button panel for instructors/hosts.

/start_session (via panel)

Purpose: Initializes a new training session. The bot begins monitoring the voice channel time and detects if the FTO or Cadet goes deaf/mute to ensure accurate logging of active training minutes. This ensures fair tracking of training time.

Message Content: A confirmation embed showing the session start time and monitoring status.

/log_training (via panel)

Purpose: Formalizes the completion of a specific training block or academy course, logging the result (Pass, Fail, Remedial) into the cadet's permanent record. This is vital for progression checks.

Message Content: A private confirmation that the training has been recorded in the cadet's file.

/view_cadet_profile [cadet]

Purpose: Pulls up a quick, clean summary of a cadet's performance metrics, total training hours, and previous evaluation results, useful for FTOs and Command staff.

Message Content: A detailed, multi-field embed displaying all relevant cadet stats.

4. Economy & Payroll

Manage your in-game paychecks and funds. Pay is automatically calculated based on your shift time. Note that command staff manage the primary payroll functions (see Supervisor Guide for those commands).

/balance

Purpose: Displays your current currency balance, along with an overview of pending pay (hours worked since last paycheck) and your current pay rate ($150/hr).

Message Content: A private ephemeral message displaying the balance details.

/transfer [user] [amount]

Purpose: Allows you to securely transfer currency to another department member for things like gear sales, reimbursement, or fines. The system requires a confirmation step to prevent accidental transfers.

Message Content: A confirmation button appears, followed by a public success announcement in the channel.

4. Disciplinary & Personnel (Sgt+ Only)

These sensitive commands are restricted to command staff (Sergeant+) to manage personnel status and implement disciplinary measures fairly and transparently.

/suspend [user] [reason]

Purpose: Suspends an officer for 3 days by default (as coded in the bot). Automatically applies the SUSPENDED role, logs the action, and sets a timed restoration. This command handles all role management automatically.

Usage Example: /suspend @OfficerJohn Failed probation period (Note: The days parameter is hardcoded to 3 days in the bot for consistency).

Message Content: A public embed confirms the suspension and includes an undo button for staff use.

/discharge [user] [reason]

Purpose: Executes a formal discharge. This command removes all previous department roles, applies the DISCHARGED role for a 24-hour hold, logs the action permanently, and notifies the relevant command channel for audit purposes.

Message Content: A public embed confirming the discharge and includes an undo button for staff use.

/infraction [user] [reason]

Purpose: Logs a minor infraction or a verbal warning into the user's personnel file without immediately changing their roles or status. Used for keeping detailed, non-critical personnel history.

Message Content: A private confirmation that the infraction was logged to the user's file.

/promote [member] [rank]

Purpose: Manually promotes a member to the specified rank role. Restricted to Administrators.

Message Content: Confirmation in the dedicated PROMOTION channel.

/demote [member] [rank]

Purpose: Manually removes the specified rank role from a member. Restricted to Administrators.

Message Content: Confirmation in the dedicated DEMOTION channel.
